# Data Model

## Database Overview

PostgreSQL + PostGIS. UUIDs generated by `pgcrypto.gen_random_uuid()`. All migrations live in `migrations/` and run automatically on server startup via `golang-migrate`.

**Current migration level:** 030 (as of schema v3)

## Tables by Migration

### 001 — `users`

Core identity table for all platform users (customers, drivers, admins).

```
id · phone_number (UK) · full_name · email · role_state · device_id
fcm_token · is_suspended · suspension_until · created_at · updated_at
```

**role_state values:** `CUSTOMER_ONLY`, `DRIVER_PENDING`, `DRIVER_ACTIVE`, `DRIVER_SUSPENDED`

---

### 002 — `driver_profiles`

One row per driver application. Created by `POST /driver/apply`.

```
id · user_id (FK users) · transport_type · vehicle_plate · license_number
date_of_birth · province · district · sector · cell · village
passenger_seats · load_capacity_kg · momo_pay_code · momo_provider
approval_status · policy_accepted · is_online · priority_tier
acceptance_rate · total_rides
```

**approval_status values:** `PENDING_REVIEW`, `ACTIVE`, `REJECTED`, `SUSPENDED`

---

### 003 — `driver_locations`

PostGIS geography point updated every time a driver sends a location update. Used as fallback when Redis GEO index is empty.

```
id · driver_id (FK) · location (geography POINT 4326) · recorded_at
```

Index: `GIST idx_driver_locations_geo`

---

### 004 — `rides`

Main ride record. Status is the single source of truth for the state machine.

```
id · customer_id · driver_id · transport_type · status
pickup_point (geography) · destination_point (geography)
pickup_address · destination_address
customer_initial_fare · agreed_fare · fare_locked_at
pickup_expired · driver_arrived_at · started_at · completed_at
cancel_reason · cancelled_by_role · created_at
```

**status values:** `SEARCHING`, `MATCHED`, `NEGOTIATING`, `CONFIRMED`, `DRIVER_EN_ROUTE`, `DRIVER_ARRIVED`, `IN_PROGRESS`, `COMPLETED`, `CANCELLED`

---

### 005 — `ride_events`

Append-only audit log. Never updated, never deleted.

```
id · ride_id · event_type · actor_role · actor_id · metadata (jsonb) · created_at
```

---

### 006 — `negotiation_rounds`

One row per offer. Tracks both proposals and responses.

```
id · ride_id · round_number · proposed_by (CUSTOMER|DRIVER)
proposed_amount · response (PENDING|ACCEPTED|DECLINED)
call_initiated · call_initiated_at · created_at
```

---

### 007 — `otp_verifications`

OTP tokens. One per request, TTL = 10 minutes.

```
id · phone · otp_hash (bcrypt) · purpose (REGISTRATION|LOGIN) · is_used · expires_at · created_at
```

---

### 008 — `device_sessions`

Security audit log of JWT sessions per device. Enables device collision detection (same device_id on multiple accounts).

```
id · user_id · device_id · jti · logged_in_at · expires_at
```

---

### 009 — `gps_anomalies` / `analytics_events`

`gps_anomalies`: Driver GPS points flagged as physically impossible (speed > 200 km/h).

`analytics_events`: Written by the Redis Stream consumer (`analytics.Consumer`). All domain events land here.

---

### 010 — Extended `users` + `driver_profiles`

Adds Rwanda administrative division columns (`province`, `district`, `sector`, `cell`, `village`) and status columns.

---

### 011 — `driver_documents`

Uploaded document metadata. Actual files live in S3/R2.

```
id · driver_id · document_type · file_url · uploaded_at
```

**document_type values:** `LICENCE_FRONT`, `VEHICLE_INSURANCE`, `VEHICLE_AUTHORIZATION`

---

### 012 — `saved_locations`

User's favorite addresses.

```
id · user_id · label · address · lat · lng · created_at
```

---

### 013 — `route_cache`

Pre-computed routes indexed by geohash6 pair + vehicle type.

```
id · cache_key (UK) · origin_geohash · dest_geohash · vehicle_type
distance_km · duration_minutes · use_count
agreed_fares (jsonb array) · avg_fare_rwf · last_used_at · created_at
```

Cache key format: `{origin_geohash6}:{dest_geohash6}:{vehicle_type}`

---

### 014 — `landmarks`

id · name · address · lat · lng · category · is_active

### 015–016 — Ride column additions

## Redis Key Reference

All key patterns are defined in `pkg/redis/redis.go`. String template notation: `{X}` = variable.

### Driver State Keys

| Key | Type | TTL | Purpose |
|---|---|---|---|
| `driver:location:{driverID}` | String (JSON) | None | Latest GPS location `{lat, lng, timestamp}` |
| `driver:location:{driverID}:history` | List | None | Recent GPS history for plausibility check |
| `driver:{driverID}:state` | String | None | `AVAILABLE` / `ON_TRIP` / `OFFLINE` |
| `driver:{driverID}:active_ride` | String | None | rideID of current active ride |
| `drivers:geo:{vehicleType}` | GEO | None | Sorted set used by GEOSEARCH for matching |

### Matching Keys

| Key | Type | TTL | Purpose |
|---|---|---|---|
| `matching:lock:{driverID}` | String | 20 s | SET NX exclusive offer lock |
| `ride:{rideID}:pending_driver` | String | match timeout (15 s) | ProfileID currently offered the ride |
| `ride:{rideID}:state` | String | None | Hot copy of ride status |

### Customer Keys

| Key | Type | TTL | Purpose |
|---|---|---|---|
| `customer:{customerID}:active_ride` | String | None | rideID pointer |

### Auth Keys

| Key | Type | TTL | Purpose |
|---|---|---|---|
| `session:{userID}:{jti}` | String | Refresh expiry | `valid` — revoked on logout |
| `ratelimit:otp:{phone}` | Integer | 1 hour | OTP attempt counter |

### Penalty Keys

| Key | Type | TTL | Purpose |
|---|---|---|---|
| `driver:penalties:{driverID}:daily_declines` | Integer | Expires at 23:59:59 UTC | Daily decline counter |
| `customer:cancels:{customerID}:daily` | Integer | EOD | Daily cancel counter |

### Route / Suggestions

| Key | Type | TTL | Purpose |
|---|---|---|---|
| `route:{cacheKey}` | String (JSON) | 24 hours | Hot route cache |
| `suggestions:{userID}` | String (JSON) | 1 hour | Location suggestions |

### Analytics / Dashboard

| Key | Type | TTL | Purpose |
|---|---|---|---|
| `analytics:events` | Stream | None | Event queue (`XADD` / `XREAD`) |
| `dashboard:snapshot` | String (JSON) | 10 seconds | KPI cache |
