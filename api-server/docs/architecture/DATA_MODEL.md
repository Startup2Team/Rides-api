# Data Model

## Database Overview

The API uses PostgreSQL with PostGIS. UUIDs are generated by `pgcrypto`.

Core table groups:

- Identity: `users`, `otp_verifications`, `device_sessions`.
- Driver: `driver_profiles`, `driver_documents`, `driver_locations`, `gps_anomalies`.
- Ride: `rides`, `ride_events`, `negotiation_rounds`.
- Location intelligence: `saved_locations`, `landmarks`, `route_cache`.
- Analytics: `analytics_events`.

## Tables

### users

Stores all platform users.

Important columns:

- `id`
- `phone_number`
- `full_name`
- `email`
- `role_state`
- `device_id`
- `fcm_token`
- `is_suspended`
- `suspension_until`

Role states used by middleware:

- `CUSTOMER_ONLY`
- `DRIVER_PENDING`
- `DRIVER_ACTIVE`
- `DRIVER_SUSPENDED`
- `ADMIN` where seeded/assigned.

### driver_profiles

One profile per user applying as a driver/rider.

Important columns:

- `user_id`
- `transport_type`
- `vehicle_plate`
- `license_number`
- `date_of_birth`
- `province`, `district`, `sector`, `cell`, `village`
- `passenger_seats`
- `load_capacity_kg`
- `momo_pay_code`
- `momo_provider`
- `approval_status`
- `policy_accepted`
- `is_online`
- `priority_tier`
- `acceptance_rate`
- `total_rides`

Approval statuses:

- `PENDING_REVIEW`
- `ACTIVE`
- `REJECTED`
- `SUSPENDED`

### driver_documents

Stores document URLs only. Files live in external object storage/CDN.

Document types:

- `LICENCE_FRONT`
- `VEHICLE_INSURANCE`
- `VEHICLE_AUTHORIZATION`

### driver_locations

Stores latest PostGIS driver location for geofence checks and fallback matching.

### rides

Main ride lifecycle record.

Important columns:

- `customer_id`
- `driver_id`
- `transport_type`
- `status`
- `pickup_point`, `destination_point`
- `pickup_address`, `destination_address`
- `customer_initial_fare`
- `agreed_fare`
- `fare_locked_at`
- `driver_arrived_at`
- `started_at`
- `completed_at`
- `pickup_expired`
- `cancel_reason`
- `cancelled_by_role`

Ride statuses:

- `SEARCHING`
- `MATCHED`
- `NEGOTIATING`
- `CONFIRMED`
- `DRIVER_EN_ROUTE`
- `DRIVER_ARRIVED`
- `IN_PROGRESS`
- `COMPLETED`
- `CANCELLED`

### ride_events

Append-only audit log for lifecycle events.

Examples:

- `ride.created`
- `ride.request_sent`
- `ride.matched`
- `ride.negotiation_started`
- `ride.fare_agreed`
- `ride.driver_en_route`
- `ride.driver_arrived`
- `ride.started`
- `ride.completed`
- `ride.cancelled`

### negotiation_rounds

Stores fare proposals per ride.

Important columns:

- `ride_id`
- `round_number`
- `proposed_by`
- `proposed_amount`
- `response`
- `call_initiated`
- `call_initiated_at`

### route_cache

Stores route data by geohash pair and vehicle type.

Important columns:

- `cache_key`
- `origin_geohash`
- `dest_geohash`
- `vehicle_type`
- `distance_km`
- `duration_minutes`
- `use_count`
- `agreed_fares`
- `avg_fare_rwf`

Public API does not expose fare suggestions yet. Data is collected for future fare intelligence.

## Redis Keys

Defined centrally in `pkg/redis`.

| Key Pattern | Purpose |
|---|---|
| `driver:location:{driverID}` | Latest driver location JSON. |
| `driver:location:{driverID}:history` | Last location updates for GPS plausibility. |
| `driver:{driverID}:state` | `AVAILABLE`, `ON_TRIP`, `OFFLINE`. |
| `driver:{driverID}:active_ride` | Active ride pointer. |
| `drivers:geo:{vehicleType}` | Redis GEO index by transport type. |
| `matching:lock:{driverID}` | SET NX lock to avoid double assignment. |
| `ride:{rideID}:pending_driver` | Driver currently offered this ride. |
| `ride:{rideID}:excluded_drivers` | Drivers skipped/declined. |
| `ride:{rideID}:state` | Hot ride state. |
| `customer:{customerID}:active_ride` | Customer active ride pointer. |
| `ride:{rideID}:offers:{role}` | Offer count cache if needed. |
| `driver:penalties:{driverID}:daily_declines` | Driver decline counter. |
| `customer:cancels:{customerID}:daily` | Customer cancellation counter. |
| `session:{userID}:{jti}` | Refresh token session status. |
| `ratelimit:otp:{phone}` | OTP rate limiting. |
| `route:{cacheKey}` | Hot route-cache JSON. |
| `suggestions:{userID}` | Cached location suggestions. |
| `analytics:events` | Redis stream for analytics. |

## Ownership Rules

- `auth` owns users during registration/login and device sessions.
- `driver` owns driver profile, documents, availability, and location updates.
- `ride` owns ride lifecycle and state transitions.
- `negotiation` owns offer rounds and fare confirmation.
- `location` owns saved locations, landmarks, route cache, fare aggregation.
- `admin` owns approval/suspension actions.
- `analytics` owns analytic reads and event consumption.
